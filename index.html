In the digital entertainment industry, automated genre classification plays a pivotal role in organizing content libraries, enhancing user experience through recommendation systems, and enabling better content discovery. Traditional genre classification using unimodal approaches, particularly based on text, has had success, especially when reliable metadata is available. In this project, we focused on a text-based multi-label classification system to predict genres of movies based on their textual metadata, particularly the movie overview.Using this approach, the model attempts to assign one or more genres (such as Action, Comedy, Drama, etc.) to each movie. Unlike unimodal fusion systems, this baseline text-only model sets the foundation for future comparisons with multimodal models (e.g., text + image fusion).

Project Goals: The primary objectives of this text-based genre classification project are:

  1. To collect and clean a movie metadata dataset using the TMDb API.
  2. perform exploratory data analysis on the overview and genre fields.
  3. implement NLP preprocessing pipelines for movie overview text.
  4. To train and evaluate a deep learning model (LSTM-based) on the genre prediction task.
  5. To establish baseline performance metrics to be compared with more advanced multimodal models.

Dataset Description

Data Source

Movie metadata was collected from the TMDb API. This included movie titles, overviews, genres, and poster paths. The data extraction process used multithreading to efficiently fetch and store metadata.

Image Data Extraction

Poster images were retrieved by combining each movie's poster_path with the base URL (https://image.tmdb.org/t/p/w500/). Posters were downloaded using Python’s requests and PIL. Faulty or missing URLs were filtered out. Verification steps included:

  1. Standardizing file names from paths
  2. Comparing expected vs. saved posters
  3. Removing entries with missing or broken image links

Data Cleaning

  1. Dropped rows with missing values
  2. Removed duplicates
  3. Ensured poster availability and consistency
  4. Final dataset: ~30,016 samples with text and image metadata

Exploratory Data Analysis (EDA)

  1. Genre Distribution: Drama, Comedy, and Action were most frequent. Rare genres like War and Western were underrepresented.
  2. Additional Taglines and Posters: Downloaded extra metadata to strengthen representation of rare genres.
  3. Genre Cleanup and Consolidation: Merged overlapping or rare genres into a new “Others” category.
  4. Genre Co-occurrence Network: Created graph connecting genres by their co-occurrence. Weakly connected genres were merged or dropped.
    5. →
  5. Heatmap Visualization: A co-occurrence heatmap supported network analysis and guided genre reduction.
  6. Multi-Label Analysis: PCA was performed to see distribution
  7. Overview Text Length: Overviews typically ranged from 30 to 150 words. Very short/long ones were reviewed.

Handling Data Imbalance

To address the challenge of genre imbalance, a multi-step strategy was employed throughout data preprocessing and augmentation. First, we observed that genres like Drama and Comedy were significantly overrepresented compared to underrepresented ones like War or Western. To mitigate this, we specifically downloaded additional metadata and posters for movies belonging to these rare categories.

Next, we refined our label space by grouping sparse or semantically overlapping genres into a new consolidated category called "Others". This helped in reducing noise and ensuring a more balanced label distribution. Moreover, genres that were disconnected in the co-occurrence network graph were either merged or removed altogether.

During model training, we employed stratified batching techniques to ensure that each batch contained a fair distribution of labels. This, combined with the earlier augmentation strategies, allowed the model to receive balanced training signals and avoid overfitting to the majority genres.

In summary, our handling of data imbalance combined augmentation, intelligent label restructuring, and balanced batch training to yield a more robust and fair multi-label classification model.

Genre Refinement and Label Strategy

  1. Tagline Retrieval: Collected additional taglines to provide more descriptive input for text modeling.
  2. Focused Augmentation: Targeted movies from low-frequency genres to address imbalance.
  3. Genre Merging: Combined similar or rare genres into “Others” to avoid fragmentation.
  4. Network Graph Construction: Analyzed genre connectivity and dropped isolated labels.
  5. Heatmap Analysis: Validated grouping decisions and measured correlation strength.

Multi-Label Binarization

After finalizing the cleaned dataset and refining the genre labels, we converted the multi-genre annotations into a machine-readable format using MultiLabelBinarizer from Scikit-learn.

Each movie can belong to multiple genres (e.g., Action, Adventure, and Comedy together), so this task is inherently multi-label. Binarization transforms the list of genres for each movie into a binary vector — where each column represents one genre, and a value of 1 indicates that the movie belongs to that genre.

This was a crucial step to enable supervised multi-label classification using models like LSTM, GRU, and DistilBERT, which expect fixed-size input and output tensors. The binary label matrix formed the ground truth used for training and evaluation.

final shape after binarization was : (21554, 18)

Additionally, this binarized format helped us:

  • Compute class-wise evaluation metrics like F1-score and precision
  • Track predicted positives vs actual positives (PP vs TP)
  • Enable genre-specific threshold tuning and co-occurrence analysis

Text-Based Movie Genre Classification

Introduction: This project focuses on multi-label genre classification using only textual data — specifically, movie overviews. Our goal was to identify one or more genres associated with a movie by analyzing its description.

Initial Approach: We began with traditional deep learning models like LSTM and GRU. Despite tuning, both models underperformed due to overfitting and weak generalization. Key limitations:

  • Dataset was relatively small for deep RNNs.
  • RNNs lacked rich contextual understanding.
  • F1 scores remained low; especially on rare genres.

Transition to Transformers: To overcome these limitations, we used DistilBERT, a lightweight transformer pretrained on large corpora. It offered semantic richness and better generalization with fewer training examples.

Model Configuration:

  • Tokenizer: distilbert-base-uncased
  • Max Length: 256 tokens
  • Loss: BCEWithLogitsLoss with class-wise positive weights
  • Optimizer: AdamW
  • Learning Rate: 2e-5
  • Batch Size: 64
  • Epochs: 25 + 5 (refinement)
  • Scheduler: CosineAnnealingLR
  • Threshold: Initially 0.5, later tuned per-genre between 0.1–0.9

Improvements Over Time:

  1. Step 1: Baseline DistilBERT with global threshold → weak precision on rare classes
  2. Step 2: Logged PP (Predicted Positives) vs TP (True Positives) to track overprediction
  3. Step 3: Saved epoch-wise models → selected best epoch using minimum PP–TP difference
  4. Step 4: Per-genre threshold optimization using F1 maximization
  5. Step 5: Used top-3 strategy: picked highest 3 genres with ≥ 0.3 probability
  6. Step 6: Additional training of 5 epochs from best checkpoint; updated if F1 improved

Final Results:

  • Macro F1 (Validation): ~0.46
  • Macro F1 (Test): ~0.40
  • Average Genres per Movie: 2.36
  • Precision & Recall: Balanced across frequent and rare genres

Multi-label Accuracy Metrics Explained:

  • Exact Match (Subset Accuracy): 0.0213
    This metric requires the model to predict all genres for a movie exactly right. It’s very strict, so scores are typically low for multi-label tasks.
  • Hamming Accuracy: 0.7136
    This measures how many individual genre labels were correctly predicted out of all possible labels, averaged across samples. It's a more lenient and interpretable score.
  • Jaccard Score (Samples Avg): 0.1562
    This compares the overlap between predicted and true labels per sample (intersection over union) and averages it. It's a good balance between exact match and Hamming.

How We Verified Genre Predictions:

  • Used confusion matrices for each genre to analyze TP/FP/FN
  • Logged and compared predicted positives with actual ground truths (PP vs TP check)
  • Manually checked random movie descriptions using model predictions vs true genres
  • Plotted ROC-AUC curves for validation and test sets
  • Other visualisations performed:
These steps ensured interpretability and gave confidence that genre predictions were not random but semantically meaningful.

Fusion-Based Movie Genre Classification

Introduction: This model leverages both textual and visual modalities by combining semantic embeddings from BERT and visual features from ResNet-18. The goal was to enhance genre classification performance by using richer multimodal data.

Why Fusion? While text-based models (like BiLSTM or DistilBERT) performed well individually, they struggled with ambiguous descriptions or missing keywords. Likewise, posters alone were not sufficient. Fusing text and image enabled the model to capture complementary signals and improved performance, especially in multi-label settings.

Dataset Summary

  • Source: Metadata + posters from TMDb API
  • Final Sample Count: ~30,000 with valid overviews and posters
  • Labels: 12 genres (multi-label), binarized using MultiLabelBinarizer
  • Image Format: All posters resized to 224x224

Preprocessing

  • Text: Cleaned overviews, BERT tokenizer (max length 128)
  • Images: Applied torchvision transforms — Resize, Normalize, ColorJitter, Affine, Horizontal Flip
  • Alignment: Ensured 1-to-1 matching between text and image data

Model Architecture

  • Text Branch: BERT → [CLS] → Linear(768→256)
  • Image Branch: ResNet-18 (pretrained) → Flatten → Linear(512→256)
  • Fusion: Concatenated [256+256] → Linear(512→128) → Dropout → Linear(128→12) → Sigmoid
  • Output: 12-dimensional multi-label prediction (one per genre)

🛠️ Training Details

  • Optimizer: Adam
  • Loss Function: BCEWithLogitsLoss
  • Epochs: 11
  • Batch Size: 32
  • Learning Rate: 1e-4
  • Dropout Rate: 0.3

Final Results

  • Validation Accuracy: ~76.7%
  • Precision: 92%
  • Recall: 87%
  • F1 Score: 92.47%
  • Test Accuracy: ~61%
  • Avg Genres per Movie: 2.36

Confusion Matrix Analysis

Genres like Comedy, Drama, and Action had high TP rates, while genres like TV Movie and Animation suffered from class imbalance and lower recall. Visual inspection confirmed that model behavior aligned with human intuition for most predictions.

Evaluation Strategy

  • Used macro/micro F1 to balance frequent and rare genres
  • Tracked Hamming accuracy and Jaccard score
  • Evaluated both exact match and per-genre thresholds
  • Manual spot-checking of top-5 predicted genres against movie posters and overviews

Inference Pipeline

  • Poster is transformed using the same torchvision preprocessing steps
  • Overview is tokenized using BERT tokenizer
  • Fused embedding is passed through the model to get genre predictions
  • Top-3 predictions with probability ≥ 0.3 are selected as final genres

🧪 Inference Script: Fusion Model Prediction

This section explains how we perform genre prediction during inference using our trained Fusion Model, which combines text (overview) and image (poster).

Step-by-Step Inference Process

  1. Model Loading:
    Loads the trained model checkpoint final_model.pt and switches to evaluation mode using model.eval().
  2. Text Preprocessing:
    Uses distilbert-base-uncased tokenizer to tokenize the overview.
    Applies padding and truncation (max length 128) and converts text to tensors.
  3. Image Preprocessing:
    The movie poster is resized to 224×224 and transformed using:
    • Resize, ToTensor, Normalize
    • Optional: ColorJitter, Affine, HorizontalFlip
  4. Input Preparation:
    Text and image tensors are moved to GPU/CPU and reshaped for batch inference.
  5. Prediction:
    Model returns raw logits → passed through sigmoid → threshold ≥ 0.3 → genre prediction.
  6. Genre Decoding:
    Final predictions are mapped back to genre names using MultiLabelBinarizer.classes_.

python fusion_inference.py \
  --image_path posters/sample.jpg \
  --overview "A young boy discovers he has magical powers and joins a hidden wizarding school."

Output

  • Shows posters of genre given

Conclusion

The journey of this project began with a focus on text-only models to classify movie genres based solely on their overviews. This allowed us to build a strong unimodal baseline using models like BiLSTM and DistilBERT. However, text descriptions alone sometimes lacked the specificity or clarity needed to differentiate closely related genres.

To overcome this, we extended our pipeline by incorporating poster image data, enabling the model to learn visual cues associated with specific genres (e.g., darker palettes for horror, vibrant tones for animation). By fusing text and image representations, the final multimodal model captured both semantic and visual context—significantly improving genre prediction accuracy and robustness.

Ultimately, this fusion approach simulates how humans perceive movie genres—by combining both the storyline and the visual design. The result is a much more reliable and generalizable genre classification system suitable for recommendation engines, content tagging, or streaming platforms.

Conclusion

The text-only model helped us understand how well genres can be predicted from descriptions alone. However, some genres were hard to identify using text. So, we fused image and text features to improve performance. The fusion model performed much better overall, capturing both visual and semantic signals for genre classification.
